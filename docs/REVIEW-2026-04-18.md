# Codebase Review 2026-04-18

**Scope**: full src/ (~22k LOC, 40 modules) + tests + docs.
**Method**: 8 parallel Explore sub-agents, one per subsystem. Top P0/P1 claims re-verified by direct file read before inclusion.

## Summary

~120 raw findings collapsed to ~40 actionable items across 4 tiers.
Strengths: test isolation, MCP/daemon unit coverage, rustls consistency, backend enum unification (partial).
Weaknesses: disk durability (no fsync/atomic rename in several paths), authorization (Telegram/MCP/IPC have no authn), god-modules (5 files >1100 LOC), schema versioning absent.

Two sub-agent claims were **withdrawn after verification**:
- `framing.rs:48` OOM pre-allocation — FALSE. Length bound check precedes `vec!`. Confirmed at lines 48–53.
- `mcp/mod.rs:42` Content-Length=0 "infinite loop" — REFRAMED. Not a loop; it's protocol desync when malformed header skipped without consuming its body.
